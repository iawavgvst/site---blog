## Установка и запуск

- Надо сделать форк репозитория и потом склонировать его себе на компьютер с помощью команды git clone (SSH). 
- После клонирования нужно зайти в папку с проектом и установить все зависимости: 1) composer install, 2) npm install.
- Используемая база данных: SQLite: чтобы ее использовать, нужно прописать в .env файле DB_CONNECTION=sqlite, DB_DATABASE=database/database.sqlite.
- Нужно сгененировать ключ приложения: php artisan key:generate.
- В некоторых случаях после переноса проекта можно очистить кэш конфигурации и маршрутов: 1) php artisan config:cache, 2) php artisan route:cache.
- В проекте имеется посев тестовых пользователей, после подключения нужно выполнить команду php artisan migrate:fresh --seed.
- Для запуска на локальном хосте нужно выполнить: 1) php srtisan serve, 2) npm run dev.
- После этого следует пройти по http://127.0.0.1:8000/ (стандартный локал).
- В routes/web.php можно посмотреть, по каким path может пройти админ (чтобы не приходить регистрацию, используйте пользователя admin из полученных тестовых пользователей).

## Дополнительно

В случае наличия конфликтов зависимости терминал предложит установить их с флагом --legacy-peer-deps: нужно выполнить эту команду.
При установке пакетов могут также возникнуть некоторые узявимости, терминал предложит получить информацю npm fund, внести исправления npm audit fix (принудительные исправления npm audit fix --force): в данном случае следует проверитьт по локальному хосту, все ли работает (отображаются ли шаблоны, не выходят ли какие-либо критические ошибки) - если да, тогда лучше не вносить исправления, тем более, принудительрные.
